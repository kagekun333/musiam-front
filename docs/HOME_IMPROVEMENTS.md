# ホーム（/）ブラッシュアップ — 改善点リスト

対象: `src/app/page.tsx` → `src/components/realm/RealmHome.tsx`（没入アトラス）
作成: 2026-06-27 / 観点: ①伯爵チャットへの動線 ②余計なものを削る ③営業(発見→購入)動線 ④モバイル/表示速度

現状の作り: 入領ゲート（「国に入る」ボタン）→ 羊皮紙のドラッグ地図 → 地方ノード（作品群）＋施設4つ＋中央の玉座。
伯爵チャットは玉座ラベル「**工房**」(`/chat`)。施設は「国の意匠/交易所/弟子入り/工房依頼」。

---

## P0 — 最優先（伯爵に辿り着けない／何をする場所か分からない）

1. **伯爵チャットの入口が分かりにくい**
   - `/chat` への導線が、地図中央の玉座ラベル「**工房**」のみ。①「工房」=会話だと分からない、②ドラッグ地図上にあり見つけにくい、③入領ゲートを通らないと到達不可。
   - → 玉座ラベルを「**伯爵と話す**」に改名（または副題追加）。さらに**ゲート／ファーストビューに常設の主要CTA「伯爵と話す」**を置き、地図に潜らせない。

2. **入領ゲートの摩擦**
   - 最初に「国に入る」を押さないと何も始まらない。来訪者の最初の一歩を一段増やしている。
   - → ゲートを残すなら**選択肢を2つ**に：「**伯爵と話す**」「**国を巡る**」。または初回以外はゲートを省略（cookie/localStorage）。

3. **「ここが何の場所か」が一言で伝わらない**
   - ゲートは「◯◯の作品でできた、ひとつの国」だけ。来訪者は“で、何ができる？”が掴めない。
   - → ファーストビューに**1行の価値提案＋主要3アクション**（話す／聴く・観る／贈る・依頼）。

## P1 — 動線とラベルの明確化（売上に直結）

4. **施設ラベルが詩的すぎて機能が読めない**
   - 「交易所」=shop、「弟子入り」=講座、「工房依頼」=法人制作、「国の意匠」=showcase。世界観は良いが、**買う・依頼する導線が見つからない**＝営業の取りこぼし。
   - → 各施設に**平易な副題**（例: 交易所〈ショップ〉/ 工房依頼〈法人・楽曲制作〉/ 弟子入り〈講座〉）。

5. **主要な収益導線が地図の隅で小さい**
   - shop・business・chat が小さなグリフのノード。300万を狙う動線としては弱い。
   - → 主要導線（話す・買う・依頼）を**ファーストビューの明示的なボタン群**にも出す（地図はその下の“体験”として残す）。

6. **モバイル体験**
   - 1300×820のドラッグ地図はスマホで操作しづらい・全体像が掴めない可能性大。
   - → スマホでは**ドラッグ地図をやめ、縦積みのカード/リスト**（伯爵と話す→巡る→買う→依頼）にフォールバック。

## P2 — 仕上げ（SEO・速度・計測）

7. **SEO**: 主要コピー・価値提案が地図(JS)依存。下部に索引はあるが、**上部にテキストの価値提案**を置くと検索/AI認知に有利（VISION_ROADMAPのGEO方針と一致）。

8. **表示速度**: 羊皮紙の `feTurbulence` フィルタ等は重い。LCP/モバイルで効く可能性。**装飾SVGの軽量化/遅延**を検討。

9. **計測**: 各入口（話す/買う/依頼/巡る）のクリックを個別イベント化し、**どこから売上に繋がるか**をPostHogで可視化（STRATEGY_10MのPhase0計測と一致）。

---

## 方向性の案（どれで進めるか選んでください）
- **案1: 軽量ヒーローを上に足す（最小・推奨）** — 今の地図はそのまま残し、上に「1行価値提案＋主要CTA（話す/聴く/贈る・依頼）」を追加。摩擦を消しつつ世界観を保つ。
- **案2: ゲートを“二択”化** — 「伯爵と話す／国を巡る」。没入感を保ったまま会話導線を最短に。
- **案3: ホームを簡素化** — 没入地図は `/realm` などへ移し、ホームはシンプルな入口に振り切る（モバイル・SEO・速度に最も効く）。

> いずれも段階的に実装可。まず案1だけでも「伯爵に辿り着けない」「何の場所か分からない」は大きく改善します。
